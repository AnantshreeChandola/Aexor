### 1.2 Users Table (Global Identity Foundation)

**Purpose**: Central user identity table that all components reference

**Schema**:
```sql
CREATE TABLE users (
  user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  full_name VARCHAR(255),
  context_tier INT NOT NULL DEFAULT 1,
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMP NOT NULL DEFAULT NOW(),
  deleted_at TIMESTAMP NULL
);

-- Indexes
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_context_tier ON users(context_tier);
CREATE INDEX idx_users_active ON users(user_id) WHERE deleted_at IS NULL;
```

**Fields**:
- `user_id`: Primary key, referenced by all component tables
- `email`: Unique identifier for authentication
- `full_name`: Display name (optional)
- `context_tier`: Consent tier (1-4, see GLOBAL_SPEC §7)
- `created_at`: User registration timestamp
- `updated_at`: Last profile update
- `deleted_at`: Soft delete timestamp (NULL = active)

**Ownership**: Auth/Registration component (to be implemented)

**Interim Ownership**: Create table now, migrate to Auth component later

**Components Referencing This**:
- ProfileStore (`preferences.user_id` → `users.user_id`)
- History (`interactions.user_id` → `users.user_id`)
- UserIntegrations (`user_integrations.user_id` → `users.user_id`)
- All components storing user-specific data

### 1.3 User Integrations Table (User-to-Tool Credential Mapping)

**Purpose**: Maps each user's integration accounts to their resolved credential vault IDs. This is a **mapping table only** — no actual credentials (OAuth tokens, API keys) are stored here. All secrets live exclusively in the encrypted credential vault (AES-256-GCM in PostgreSQL).

**Schema**:
```sql
CREATE TABLE user_integrations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(user_id),
  tool_id VARCHAR(128) NOT NULL,           -- e.g., "google.calendar" (references PluginRegistry)
  account_name VARCHAR(128) NOT NULL,      -- e.g., "work", "personal", "default"
  credential_id VARCHAR(255) NOT NULL,     -- resolved credential ID for vault (e.g., "gcal_user_u-123_work")
  mcp_server VARCHAR(128) NOT NULL,        -- e.g., "google-calendar" (from PluginRegistry)
  status VARCHAR(20) NOT NULL DEFAULT 'active', -- active | revoked | expired
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMP NOT NULL DEFAULT NOW(),

  UNIQUE (user_id, tool_id, account_name)  -- one account_name per tool per user
);

-- Indexes
CREATE INDEX idx_user_integrations_user ON user_integrations(user_id);
CREATE INDEX idx_user_integrations_tool ON user_integrations(tool_id);
CREATE INDEX idx_user_integrations_active ON user_integrations(user_id, tool_id) WHERE status = 'active';
```

**Fields**:
- `user_id`: Which user owns this integration account (FK → `users`)
- `tool_id`: Which tool this account is for (logical FK → PluginRegistry, e.g., `google.calendar`)
- `account_name`: User-facing label for the account (e.g., "work", "personal"). Unique per user+tool pair
- `credential_id`: The resolved credential identifier used to look up the actual secret in the encrypted vault. Generated by interpolating PluginRegistry's credential template with user-specific values
- `mcp_server`: The MCP server identifier (copied from PluginRegistry at creation time, e.g., `google-calendar`)
- `status`: `active` (usable), `revoked` (user disconnected), `expired` (token expired, needs re-auth)

**Security constraints**:
- This table NEVER stores actual credentials (OAuth tokens, API keys, secrets)
- `credential_id` is an opaque string reference — the actual secret is decrypted from the credential vault at execution time
- No credential values in logs — only `credential_id` may be logged

**How it's used**:
1. **Admin/user onboarding**: When a user connects a tool (e.g., links their Google account), a row is created with the resolved `credential_id`. The actual OAuth token is stored in the encrypted credential vault (AES-256-GCM)
2. **Planner**: Queries UserIntegrations to find which `credential_id` to embed in the plan for a given user + tool
3. **ExecuteOrchestrator**: Uses `credential_id` to decrypt the credential from the vault at MCP invocation time
4. **Pre-execution validation**: Checks that the user has an active integration for all tools referenced in a plan

**Ownership**: Shared infrastructure (co-owned with PluginRegistry)

**Components Referencing This**:
- Planner (resolve credential_id for plan steps)
- ExecuteOrchestrator (decrypt credentials from vault at MCP invocation time)
- PluginRegistry (pre-execution validation checks active integrations)
- Any future admin UI for managing user connections

## 2. Authentication & Authorization

### 2.1 Auth Middleware

**Purpose**: Extract user identity and consent tier from requests

**Location**: `shared/middleware/auth.py`

**Interface**:
```python
# Request state (populated by middleware)
request.state.user_id: UUID       # Authenticated user ID
request.state.context_tier: int   # Consent tier (1-4)
request.state.email: str          # User email
```

**Implementation Phases**:

#### Phase 1: MVP (Header-Based Auth)
For rapid development, use simple header-based auth:

```python
# shared/middleware/auth.py
from fastapi import Request, HTTPException
from uuid import UUID

async def auth_middleware(request: Request, call_next):
    """
    MVP: Extract user context from headers.
    TODO: Replace with JWT validation in Phase 2.
    """
    user_id = request.headers.get("X-User-ID")
    context_tier = request.headers.get("X-Context-Tier", "1")

    if not user_id:
        raise HTTPException(status_code=401, detail="Missing X-User-ID header")

    # Add to request state
    request.state.user_id = UUID(user_id)
    request.state.context_tier = int(context_tier)
    request.state.email = request.headers.get("X-User-Email", "dev@example.com")

    response = await call_next(request)
    return response
```

**Usage in Components**:
```python
# components/ProfileStore/api/routes.py
from fastapi import Request

@app.get("/preferences/{preference_key}")
async def get_preference(request: Request, preference_key: str):
    user_id = request.state.user_id           # UUID
    context_tier = request.state.context_tier  # int

    # Enforce consent tier
    if context_tier < 2:
        return {"error": "CONSENT_DENIED"}

    # ... fetch preference
```

---

#### Phase 2: Production (JWT-Based Auth)
For production, implement full JWT validation:

```python
# shared/middleware/auth.py
from fastapi import Request, HTTPException
from jose import jwt, JWTError
import os

JWT_SECRET = os.getenv("JWT_SECRET")
JWT_ALGORITHM = "HS256"

async def auth_middleware(request: Request, call_next):
    """
    Production: Validate JWT token and extract claims.
    """
    auth_header = request.headers.get("Authorization")
    if not auth_header or not auth_header.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Missing or invalid Authorization header")

    token = auth_header.split(" ")[1]

    try:
        payload = jwt.decode(token, JWT_SECRET, algorithms=[JWT_ALGORITHM])
        request.state.user_id = UUID(payload["sub"])
        request.state.context_tier = payload.get("context_tier", 1)
        request.state.email = payload.get("email")
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

    response = await call_next(request)
    return response
```

**JWT Claims Structure**:
```json
{
  "sub": "user-uuid",
  "email": "user@example.com",
  "context_tier": 3,
  "exp": 1735430400,
  "iat": 1735344000
}
```

---

### 2.2 Migration Path: Phase 1 → Phase 2

**Phase 1 (MVP)**:
- Use header-based auth (`X-User-ID`, `X-Context-Tier`)
- No JWT validation
- Development only (not production-safe)

**Phase 2 (Production)**:
- Replace with JWT validation
- Zero code changes in components (same `request.state` interface)
- Only middleware implementation changes

**Timeline**: Implement Phase 1 now, Phase 2 before multi-user deployment

## 3. Encryption Services

### 3.1 Shared Encryption Utilities

**Purpose**: Reusable encryption/decryption for sensitive data

**Location**: `shared/security/encryption.py`

**Algorithm**: AES-256-GCM (AEAD)

**Implementation**:
```python
# shared/security/encryption.py
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os
import base64

class EncryptionService:
    """
    Shared encryption service using AES-256-GCM.
    Thread-safe and reusable across components.
    """
    def __init__(self, key: bytes = None):
        if key is None:
            # Load from environment
            key_b64 = os.getenv("ENCRYPTION_KEY")
            if not key_b64:
                raise ValueError("ENCRYPTION_KEY environment variable not set")
            key = base64.b64decode(key_b64)

        if len(key) != 32:
            raise ValueError("Encryption key must be 32 bytes (256 bits)")

        self.cipher = AESGCM(key)

    def encrypt(self, plaintext: str) -> str:
        """
        Encrypt plaintext and return base64-encoded ciphertext.
        Format: {iv}:{ciphertext} (tag included in ciphertext by GCM)
        """
        iv = os.urandom(12)  # 96-bit IV for GCM
        ciphertext = self.cipher.encrypt(iv, plaintext.encode(), None)

        return f"{base64.b64encode(iv).decode()}:{base64.b64encode(ciphertext).decode()}"

    def decrypt(self, encrypted: str) -> str:
        """
        Decrypt base64-encoded ciphertext.
        Format: {iv}:{ciphertext}
        """
        iv_b64, ciphertext_b64 = encrypted.split(":")
        iv = base64.b64decode(iv_b64)
        ciphertext = base64.b64decode(ciphertext_b64)

        plaintext = self.cipher.decrypt(iv, ciphertext, None)
        return plaintext.decode()
```

**Key Management**:
```bash
# Generate encryption key (32 bytes)
python -c "import secrets, base64; print(base64.b64encode(secrets.token_bytes(32)).decode())"

# Store in environment
export ENCRYPTION_KEY="<base64-encoded-key>"
```

**Usage in Components**:
```python
# components/ProfileStore/adapters/encryption.py
from shared.security.encryption import EncryptionService

class ProfileStoreEncryptionAdapter:
    def __init__(self):
        self.encryption_service = EncryptionService()  # Uses global key

    def encrypt_preference(self, value: str) -> str:
        return self.encryption_service.encrypt(value)

    def decrypt_preference(self, encrypted: str) -> str:
        return self.encryption_service.decrypt(encrypted)
```

**Components Using This**:
- ProfileStore (sensitive preferences)
- Any component storing PII (passport numbers, health data, etc.)

## 5. Database Migrations

### 5.1 Migration Framework

**Technology**: Alembic (SQLAlchemy migrations)

**Location**: `/migrations/`

**Setup**:
```bash
# Initialize Alembic
alembic init migrations

# Edit alembic.ini
sqlalchemy.url = postgresql://localhost:5432/personal_agent
```

**Directory Structure**:
```
migrations/
├── alembic.ini           # Alembic configuration
├── env.py                # Migration environment
├── script.py.mako        # Migration template
└── versions/
    ├── 001_create_users_table.py
    ├── 002_create_preferences_table.py
    ├── 003_create_user_integrations_table.py
    └── ...
```

**Creating Migrations**:
```bash
# Generate migration
alembic revision -m "create users table"

# Apply migrations
alembic upgrade head

# Rollback
alembic downgrade -1
```

**Example Migration** (`001_create_users_table.py`):
```python
"""create users table

Revision ID: 001
Create Date: 2025-12-26
"""
from alembic import op
import sqlalchemy as sa
from sqlalchemy.dialects.postgresql import UUID

def upgrade():
    op.create_table(
        'users',
        sa.Column('user_id', UUID(as_uuid=True), primary_key=True, server_default=sa.text('gen_random_uuid()')),
        sa.Column('email', sa.String(255), unique=True, nullable=False),
        sa.Column('full_name', sa.String(255)),
        sa.Column('context_tier', sa.Integer, nullable=False, server_default='1'),
        sa.Column('created_at', sa.TIMESTAMP, nullable=False, server_default=sa.text('NOW()')),
        sa.Column('updated_at', sa.TIMESTAMP, nullable=False, server_default=sa.text('NOW()')),
        sa.Column('deleted_at', sa.TIMESTAMP)
    )
    op.create_index('idx_users_email', 'users', ['email'])
    op.create_index('idx_users_context_tier', 'users', ['context_tier'])

def downgrade():
    op.drop_table('users')
```

## 9. Security Best Practices

1. **Never commit secrets** to git
2. **Use `.env` file** (gitignored) for local development
3. **Use secret manager** (AWS Secrets Manager, etc.) for production
4. **Rotate encryption keys** periodically (if multi-user)
5. **Use strong random keys** (32+ bytes)
6. **Enable database SSL/TLS** in production
